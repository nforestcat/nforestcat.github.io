운영체제와 리눅스 기초 정리

운영체제와 리눅스 기초 수업을 들으면서 정리한 내용이다. 전체 흐름은 다음과 같다.

운영체제 소개
→ 프로세스
→ 메모리
→ 파일 시스템
→ 리눅스 입문
→ 리눅스 파일 시스템
→ 리눅스 명령어
→ 프로세스와 네트워크

1. 운영체제 소개

운영체제, 즉 OS는 컴퓨터 하드웨어와 사용자가 실행하는 프로그램 사이에서 자원을 관리하는 핵심 소프트웨어다.

사용자 / 프로그램
        ↓
운영체제
        ↓
하드웨어

예를 들어 사용자가 Python 파일을 실행하면 운영체제는 다음과 같은 일을 한다.

1. 프로그램을 실행한다.
2. 필요한 파일을 디스크에서 읽는다.
3. 프로그램을 메모리에 올린다.
4. CPU가 실행할 수 있도록 스케줄링한다.
5. 입력과 출력을 처리한다.
6. 프로그램이 끝나면 사용한 자원을 회수한다.

즉, 운영체제는 CPU, 메모리, 파일, 장치, 네트워크 같은 컴퓨터 자원을 관리하고 프로그램이 안전하게 실행될 수 있도록 도와준다.

운영체제의 주요 기능

1. 프로세스 관리
2. 메모리 관리
3. 파일 시스템 관리
4. 입출력 장치 관리
5. 사용자 인터페이스 제공
6. 보안과 권한 관리

커널과 셸

운영체제 구조는 다음처럼 이해할 수 있다.

사용자
  ↓
응용 프로그램
  ↓
Shell 또는 GUI
  ↓
Kernel
  ↓
하드웨어

커널은 운영체제의 핵심이다. CPU, 메모리, 프로세스, 파일 시스템, 장치, 네트워크를 직접 관리한다.

셸은 사용자의 명령어를 해석해서 커널에 전달하는 프로그램이다.

예를 들어 터미널에서 다음 명령어를 입력하면:

ls

흐름은 다음과 같다.

사용자 입력: ls
→ Shell이 명령어 해석
→ Kernel에 파일 목록 요청
→ 결과 출력

2. 프로세스

프로세스는 실행 중인 프로그램이다.

프로그램 = 실행 전 파일
프로세스 = 실행 중인 프로그램

예를 들어 program.py라는 파일이 디스크에 저장되어 있을 때는 프로그램이다.

program.py

하지만 다음처럼 실행하면 운영체제가 이 프로그램을 메모리에 올리고 CPU가 실행할 수 있게 만든다.

python program.py

이 순간부터는 프로세스가 된다.

프로그램, 프로세스, 스레드

구분 의미
프로그램 특정 작업을 수행하는 명령어들의 모음
프로세스 실행 중인 프로그램
스레드 프로세스 안에서 실행되는 흐름 단위

하나의 프로세스 안에는 하나 이상의 스레드가 존재할 수 있다.

Chrome 프로세스
├─ 화면 렌더링 스레드
├─ 네트워크 요청 스레드
├─ 사용자 입력 처리 스레드
└─ 확장 프로그램 처리 스레드

프로세스와 스레드의 차이

구분 프로세스 스레드
실행 단위 독립된 실행 단위 프로세스 내부 실행 흐름
메모리 독립적인 메모리 공간 사용 같은 프로세스 메모리 공유
생성 비용 상대적으로 큼 상대적으로 작음
안정성 하나가 죽어도 다른 프로세스에 영향이 적음 한 스레드 문제가 전체 프로세스에 영향 가능

핵심은 다음과 같다.

프로세스 = 독립된 실행 공간
스레드 = 프로세스 안의 실행 흐름

3. 프로세스의 메모리 구조

프로세스가 실행되면 메모리 안에 여러 영역을 가진다.

Code 영역
Data 영역
BSS 영역
Heap 영역
Stack 영역
영역 의미
Code 영역 CPU가 실행할 명령어가 저장되는 영역
Data 영역 전역 변수, 정적 변수가 저장되는 영역
BSS 영역 초기화되지 않은 전역 변수, 정적 변수가 저장되는 영역
Heap 영역 실행 중 동적으로 생성되는 데이터가 저장되는 영역
Stack 영역 함수 호출, 매개변수, 지역 변수, 복귀 주소가 저장되는 영역

Code 영역

Code 영역은 실행할 명령어가 저장되는 곳이다. Text 영역이라고도 부른다.

예를 들어 다음 함수의 실행 명령은 Code 영역과 관련된다.

def add(a, b):
    return a + b

Data 영역

Data 영역은 프로그램 시작부터 종료까지 유지되는 값들이 저장되는 공간이다.

예를 들어 전역 변수는 Data 영역과 관련된다.

count = 0

def increase():
    global count
    count += 1

Heap 영역

Heap 영역은 실행 중에 동적으로 만들어지는 데이터가 저장되는 곳이다.

Python에서는 리스트, 딕셔너리, 객체 같은 데이터가 Heap 영역과 관련된다고 이해할 수 있다.

numbers = [1, 2, 3, 4, 5]
person = {"name": "Kim"}

Stack 영역

Stack 영역은 함수 호출과 관련된 임시 데이터가 저장되는 곳이다.

def add(a, b):
    result = a + b
    return result

여기서 a, b, result는 함수 실행 중 필요한 임시 데이터다. 함수 호출이 끝나면 해당 Stack 영역도 사라진다.

Stack은 후입선출 구조다.

LIFO = Last In, First Out
마지막에 들어간 것이 먼저 나온다.

4. 프로세스의 상태

프로세스는 실행되는 동안 여러 상태를 오간다.

생성
준비
실행
대기
종료
상태 의미
생성 프로세스가 만들어지는 단계
준비 CPU만 받으면 실행할 수 있는 상태
실행 CPU를 할당받아 실제로 실행 중인 상태
대기 입출력이나 특정 이벤트를 기다리는 상태
종료 실행이 끝난 상태

전체 흐름은 다음과 같다.

생성
 ↓
준비
 ↓
실행
 ↓
종료

하지만 실행 중에 파일 읽기, 네트워크 응답, 키보드 입력 같은 대기 상황이 생기면 다음 흐름이 된다.

실행
 ↓
대기
 ↓
준비
 ↓
실행

5. 멀티 프로세싱과 멀티 스레딩

멀티 프로세싱

멀티 프로세싱은 여러 프로세스를 실행해서 작업을 나누는 방식이다.

프로세스 A 실행
프로세스 B 실행
프로세스 C 실행

각 프로세스는 독립적인 메모리 공간을 가진다. 따라서 안정성은 상대적으로 높지만, 생성 비용과 메모리 사용량이 크다.

멀티 스레딩

멀티 스레딩은 하나의 프로세스 안에서 여러 스레드가 작업을 나누는 방식이다.

프로세스
├─ 스레드 1
├─ 스레드 2
└─ 스레드 3

스레드들은 같은 프로세스의 메모리를 공유한다. 따라서 생성 비용이 작고 빠르지만, 하나의 스레드 문제가 전체 프로세스에 영향을 줄 수 있다.

6. 프로세스 스케줄링

프로세스 스케줄링은 운영체제가 어떤 프로세스에게 CPU를 줄지 결정하는 작업이다.

Ready Queue
[P1, P2, P3, P4]

CPU는 제한된 자원이기 때문에 운영체제가 실행 순서를 정해야 한다.

P1 실행
→ P2 실행
→ P3 실행
→ 다시 P1 실행

사용자 입장에서는 여러 프로그램이 동시에 실행되는 것처럼 보이지만, 실제로는 운영체제가 CPU 사용 시간을 잘게 나누어 배분한다.

7. 교착상태와 기아상태

교착상태

교착상태는 여러 프로세스가 서로 자원을 기다리면서 아무도 진행하지 못하는 상태다.

예를 들어:

프로세스 A는 프린터를 가지고 있고, 스캐너를 기다림
프로세스 B는 스캐너를 가지고 있고, 프린터를 기다림

이 경우 두 프로세스 모두 서로가 가진 자원을 기다리므로 멈춘다.

교착상태 = 서로 자원을 기다리며 멈춘 상태

기아상태

기아상태는 특정 프로세스가 계속 실행 기회를 얻지 못하는 상태다.

예를 들어 우선순위가 높은 프로세스가 계속 들어오면, 우선순위가 낮은 프로세스는 계속 밀릴 수 있다.

기아상태 = 특정 프로세스가 자원이나 CPU를 계속 배정받지 못하는 상태

8. 메모리 관리

메모리 관리는 프로그램들이 잘 실행될 수 있도록 메모리를 할당하고 해제하는 작업이다.

운영체제는 여러 프로그램이 동시에 실행될 수 있도록 메모리 공간을 나누어 주고, 프로그램이 끝나면 해당 공간을 회수한다.

프로그램 실행
→ 메모리 할당

프로그램 종료
→ 메모리 해제

메모리 주소

메모리는 여러 칸으로 나누어져 있고, 각 칸에는 주소가 있다.

[ ] [ ] [ ] [ ] [ ] [ ] [ ]
 0   1   2   3   4   5   6

CPU는 이 주소를 기준으로 데이터를 찾는다.

9. 연속 할당과 불연속 할당

연속 할당

연속 할당은 하나의 프로그램에게 연속된 메모리 공간을 주는 방식이다.

[ A ][ A ][ A ][ A ][ ][ ][ ]

장점은 단순하다는 점이다. 하지만 연속된 큰 공간이 필요하기 때문에 외부 단편화 문제가 생길 수 있다.

불연속 할당

불연속 할당은 하나의 프로그램을 여러 조각으로 나누어 떨어진 메모리 공간에 배치하는 방식이다.

[ A ][ ][ A ][ ][ A ][ A ][ ]

장점은 빈 공간을 더 유연하게 사용할 수 있다는 점이다. 단점은 어느 조각이 어디에 있는지 관리해야 해서 복잡하다는 점이다.

10. 외부 단편화와 내부 단편화

외부 단편화

외부 단편화는 빈 공간은 충분하지만, 그 공간들이 여기저기 흩어져 있어서 사용할 수 없는 문제다.

[사용][빈 2칸][사용][빈 3칸][사용][빈 2칸]

빈 공간을 모두 합치면 7칸이지만, 연속된 5칸이 필요한 프로그램은 들어갈 수 없다.

외부 단편화
= 빈 공간이 조각조각 흩어져 큰 연속 공간을 만들지 못하는 문제

내부 단편화

내부 단편화는 할당받은 공간 내부에서 실제로 쓰지 않는 공간이 낭비되는 문제다.

필요한 공간: 3칸
할당받은 공간: 4칸
낭비: 1칸

내부 단편화
= 할당받은 공간 안에서 실제 사용하지 않는 공간이 생기는 문제

11. 가상 메모리

가상 메모리는 실제 메모리보다 더 큰 메모리를 사용하는 것처럼 보이게 하는 기술이다.

프로그램이 보는 메모리 = 가상 메모리
실제 장치에 있는 메모리 = 물리 메모리

프로그램은 자신만의 독립적인 메모리 공간을 가진 것처럼 동작한다. 운영체제와 하드웨어가 가상 주소를 실제 물리 주소로 변환해준다.

가상 메모리를 사용하는 이유는 다음과 같다.

1. 프로그램마다 독립적인 메모리 공간을 제공하기 위해
2. 실제 RAM보다 큰 프로그램도 실행할 수 있게 하기 위해
3. 프로세스 간 메모리 보호를 위해

12. 페이징과 세그멘테이션

페이징

페이징은 메모리를 같은 크기의 작은 블록으로 나누어 관리하는 방식이다.

가상 메모리 → 페이지
물리 메모리 → 프레임

페이지는 물리 메모리에 연속적으로 올라갈 필요가 없다.

페이지 0 → 프레임 3
페이지 1 → 프레임 0
페이지 2 → 프레임 5

페이징은 외부 단편화를 줄일 수 있지만, 마지막 페이지에서 내부 단편화가 생길 수 있다.

세그멘테이션

세그멘테이션은 메모리를 의미 있는 단위로 나누는 방식이다.

Code Segment
Data Segment
Heap Segment
Stack Segment

페이징과 세그멘테이션의 차이는 다음과 같다.

구분 페이징 세그멘테이션
나누는 기준 고정 크기 의미 단위
조각 크기 모두 같음 서로 다를 수 있음
장점 외부 단편화 감소 프로그램 구조와 잘 맞음
단점 내부 단편화 가능 외부 단편화 가능

13. 파일과 파일 시스템

파일은 컴퓨터에서 정보를 담는 논리적인 저장 단위다.

memo.txt
main.py
image.png
data.csv

파일은 내용뿐만 아니라 메타데이터도 가진다.

파일 이름
유형
저장 위치
파일 크기
권한
생성 시간
수정 시간

파일 시스템은 파일을 저장하고 검색하고 관리하기 위한 체계다.

파일 시스템
= 파일을 쉽게 접근할 수 있도록 관리하는 체제

14. 디스크 할당

파일 시스템에서는 파일을 디스크 어디에 저장할지 결정해야 한다.

대표적인 파일 할당 방식은 다음과 같다.

1. 연속 할당
2. 연결 할당
3. FAT 방식
4. 색인 할당

연속 할당

파일을 연속된 블록에 저장한다.

[ A ][ A ][ A ][ A ][ ][ ][ ]

장점은 빠르다는 점이다. 단점은 외부 단편화가 생길 수 있고, 파일이 커질 때 대응하기 어렵다는 점이다.

연결 할당

파일의 각 블록이 다음 블록의 위치를 가리키도록 저장한다.

16 → 9 → 12 → 31

장점은 연속된 공간이 없어도 저장할 수 있다는 점이다. 단점은 직접 접근이 어렵고, 중간 연결이 손상되면 이후 데이터를 읽기 어렵다는 점이다.

FAT 방식

FAT는 File Allocation Table의 약자다. 연결 정보를 별도의 표에 저장한다.

블록 16 → 다음 9
블록 9  → 다음 12
블록 12 → 다음 31
블록 31 → 끝

색인 할당

파일마다 인덱스 블록을 두고, 그 안에 파일이 사용하는 블록 번호를 저장한다.

파일 A의 인덱스 블록
[16, 9, 12, 31]

15. inode와 링크

inode는 파일의 실제 정보를 담는 구조다.

inode에는 다음 정보가 들어간다.

파일 유형
파일 접근 권한
파일 시간 정보
파일이 저장된 블록 위치

파일명은 실제 데이터 자체가 아니라 inode를 가리키는 이름이라고 볼 수 있다.

memo.txt → inode 100 → 실제 데이터

하드 링크

하드 링크는 같은 inode를 가리키는 또 다른 파일명이다.

original.txt ─┐
              ├→ inode 100 → 실제 데이터
hard.txt     ─┘

원본 파일명을 삭제해도 다른 하드 링크가 있으면 데이터는 남아 있다.

심볼릭 링크

심볼릭 링크는 원본 파일의 경로를 가리키는 바로가기다.

soft.txt → original.txt → inode 100 → 실제 데이터

원본이 삭제되면 심볼릭 링크는 깨진다.

16. 마운트

마운트는 저장장치나 파일 시스템을 리눅스의 특정 디렉터리에 연결하는 작업이다.

리눅스는 Windows처럼 C:\, D:\ 드라이브가 따로 보이는 구조가 아니다. 모든 것은 / 아래의 트리 구조에 붙는다.

USB 저장장치 → /mnt/usb

마운트하면 사용자는 /mnt/usb 디렉터리로 들어가 USB 내용을 볼 수 있다.

cd /mnt/usb
ls

17. 리눅스란?

리눅스는 리누스 토발즈가 만든 오픈소스 운영체제다. 서버, 클라우드, 컨테이너, 개발환경에서 많이 사용된다.

웹 서버
DB 서버
Docker 컨테이너
클라우드 서버
AI/데이터 분석 서버
임베디드 장비

리눅스 구조

응용 프로그램
   ↓
Shell
   ↓
Kernel
   ↓
하드웨어

리눅스의 특징

유닉스 기반
멀티 유저 지원
멀티 태스킹 지원
CLI와 GUI 모두 지원
강력한 네트워킹 기능

18. 리눅스 기본 명령어

현재 위치 확인

pwd

파일 목록 보기

ls
ls -l
ls -al

디렉터리 이동

cd 폴더명
cd ..
cd ~
cd /

디렉터리 만들기

mkdir test
mkdir -p project/src/components

파일 만들기

touch memo.txt

파일 내용 보기

cat memo.txt

파일 복사

cp memo.txt backup.txt

파일 이동 또는 이름 변경

mv memo.txt note.txt

파일 삭제

rm memo.txt

디렉터리 삭제

rmdir test
rm -r test
rm -rf test

rm -rf는 확인 없이 강제로 삭제하기 때문에 조심해야 한다.

19. 리눅스 파일 권한

리눅스에서는 파일마다 소유자와 권한이 있다.

ls -al

예시:

-rw-r--r-- 1 user user 1234 May 28 10:30 memo.txt

권한 부분은 다음처럼 나눌 수 있다.

-   rw-   r--   r--
│    │     │     │
│    │     │     └─ others 권한
│    │     └─────── group 권한
│    └───────────── owner 권한
└────────────────── 파일 종류

권한의 종류는 다음과 같다.

r = read = 읽기
w = write = 쓰기
x = execute = 실행

권한 변경은 chmod를 사용한다.

chmod 644 memo.txt
chmod 755 script.sh
chmod 700 private_dir

숫자 권한은 다음 값을 더해서 만든다.

r = 4
w = 2
x = 1

자주 쓰는 권한은 다음과 같다.

644 = 일반 파일
755 = 실행 파일 또는 일반 디렉터리
600 = 개인 비밀 파일
700 = 개인 디렉터리
777 = 모두 허용, 보안상 주의

20. 정규표현식

정규표현식은 특정한 규칙을 가진 문자열 패턴을 표현하는 방식이다.

일반 검색 = 특정 단어 찾기
정규표현식 검색 = 특정 규칙을 만족하는 문자열 찾기

자주 쓰는 메타 문자는 다음과 같다.

표현 의미
^ 문자열의 시작
$ 문자열의 끝
. 아무 문자 하나
\d 숫자
\D 숫자가 아닌 것
\w 알파벳, 숫자, 언더스코어
\W \w가 아닌 것
\s 공백 문자
\S 공백이 아닌 문자

수량자는 다음과 같다.

표현 의미
? 0개 또는 1개
* 0개 이상
+ 1개 이상
{n} n개
{n,m} n개 이상 m개 이하
{n,} n개 이상

예시:

^010(\D?\d{4}){2}

의미:

010으로 시작
구분문자가 있을 수도 있고 없을 수도 있음
숫자 4개 패턴이 2번 반복

21. grep

grep은 파일 안에서 특정 패턴이나 문자열을 찾는 명령어다.

grep [option] [pattern] [파일명]

예:

grep error app.log

주요 옵션은 다음과 같다.

옵션 의미
-i 대소문자 구분 없이 검색
-v 패턴과 일치하지 않는 행 출력
-c 패턴과 일치하는 행 개수 출력
-w 단어 단위로 매칭

예시:

grep -i error app.log
grep -v "^#" config.conf
grep -c elice hello.txt
grep "[0-9]" hello.txt

22. 리다이렉션

리다이렉션은 표준 입력, 표준 출력, 표준 오류의 흐름을 바꾸는 기능이다.

표준 스트림은 다음 세 가지다.

이름 의미 번호
stdin 표준 입력 0
stdout 표준 출력 1
stderr 표준 오류 2

출력 리다이렉션

ls > result.txt

ls 결과를 화면에 출력하지 않고 result.txt에 저장한다. 기존 내용은 덮어쓴다.

이어쓰기

ls >> result.txt

기존 파일 내용 뒤에 이어 쓴다.

입력 리다이렉션

cat < hello.txt

파일 내용을 명령어의 입력으로 사용한다.

오류 리다이렉션

ls 없는파일 2> error.txt

오류 메시지를 error.txt에 저장한다.

23. 파이프

파이프는 앞 명령어의 출력을 뒤 명령어의 입력으로 전달하는 기능이다.

명령어1 | 명령어2

예:

head a.txt | grep [0-9]

흐름은 다음과 같다.

1. a.txt의 첫 10줄을 출력
2. 그 결과를 grep에 전달
3. 숫자가 포함된 행만 출력

자주 쓰는 예시는 다음과 같다.

ls | grep ".txt"
ps -ef | grep python
cat app.log | grep error

24. 프로세스 명령어

ps

ps는 현재 실행 중인 프로세스 목록을 보는 명령어다.

ps
ps -ef
ps -aux

주요 옵션은 다음과 같다.

옵션 의미
-e 현재 실행 중인 모든 프로세스 출력
-f 자세한 정보 출력
-a 전체 사용자의 프로세스 출력
-u 사용자와 시작 시간 등 출력
-x 터미널 제어 없이 실행 중인 프로세스 출력

특정 프로세스를 찾을 때는 grep과 같이 쓴다.

ps -ef | grep python
ps -aux | grep mysql

kill

kill은 프로세스에 시그널을 보내는 명령어다. 보통 프로세스를 종료할 때 사용한다.

kill PID
kill -9 PID
명령 의미
kill PID 정상 종료 요청
kill -9 PID 강제 종료

처음부터 kill -9를 쓰기보다는 일반 종료 요청 후, 종료되지 않을 때 강제 종료를 쓰는 것이 좋다.

25. job

job은 현재 터미널 세션에서 실행한 작업 단위다.

프로세스 = 운영체제 전체에서 실행 중인 프로그램
job = 현재 터미널에서 관리하는 작업

Foreground

터미널 앞에서 실행되는 작업이다.

python server.py

이 경우 터미널이 해당 프로그램에 붙잡힌다.

Background

명령어 뒤에 &를 붙이면 백그라운드에서 실행된다.

python server.py &
sleep 100 &

현재 터미널의 job 목록은 다음 명령어로 확인한다.

jobs

job을 종료할 때는 다음처럼 할 수 있다.

kill %1

26. at과 cron

at과 cron은 자동 작업 예약과 관련된 명령어다.

at = 한 번만 실행할 작업 예약
cron = 반복 실행할 작업 예약

at

at now + 1 minutes

입력 후 실행할 명령어를 작성하고 Ctrl + D로 종료한다.

작업 목록 확인:

atq

예약 작업 삭제:

atrm 작업번호

cron

cron은 반복 작업을 예약할 때 쓴다.

crontab -e

현재 등록된 cron 확인:

crontab -l

cron 형식은 다음과 같다.

분 시 일 월 요일 명령어

예시:

0 3 * * * /home/user/backup.sh

의미:

매일 새벽 3시에 backup.sh 실행

5분마다 실행:

*/5 * * * * /home/user/check.sh

27. SSH

SSH는 Secure Shell의 약자다. 네트워크를 통해 원격 서버에 안전하게 접속하는 방법이다.

ssh 사용자명@서버주소

예:

ssh ubuntu@192.168.0.10

포트를 지정할 수도 있다.

ssh -p 22 ubuntu@192.168.0.10

기본 SSH 포트는 보통 22번이다.

SSH는 접속하는 쪽과 접속을 받아주는 쪽이 있다.

SSH Client = 접속을 시도하는 내 컴퓨터
SSH Server = 접속을 받아주는 원격 서버

Ubuntu에서 SSH 서버를 설치하려면 다음처럼 할 수 있다.

sudo apt install openssh-server
sudo systemctl start ssh
sudo systemctl status ssh

SSH 포트 확인 예시:

ss -tulnp | grep ssh

이번 수업 전체 흐름은 다음처럼 이어진다.

= 실행 중인 프로그램

= 명령어 기반으로 리눅스를 조작하는 환경

정규표현식과 grep
= 문자열을 패턴으로 검색하는 방법

리다이렉션과 파이프

= 실행 중인 프로세스와 작업을 관리하는 방법

at, cron
= 작업을 자동으로 실행하는 방법

SSH
= 원격 리눅스 서버에 접속하는 방법

마무리

운영체제와 리눅스를 공부할 때 중요한 것은 단순히 명령어를 외우는 것이 아니라, 명령어가 운영체제의 어떤 기능과 연결되는지 이해하는 것이다.

ps, kill
→ 프로세스 관리

chmod, chown
→ 권한 관리

mkdir, touch, rm
→ 파일 시스템 관리

grep, pipe, redirection
→ CLI 기반 데이터 처리

cron, at
→ 작업 스케줄링

ssh
→ 네트워크와 원격 서버 관리

리눅스는 처음에는 명령어가 많아 보여서 어렵지만, 결국 핵심은 다음 문장으로 정리할 수 있다.
